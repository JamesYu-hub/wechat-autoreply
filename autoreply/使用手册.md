# 微信自动回复使用手册

项目目录示例：

```text
/path/to/summaryassist
```

在 Terminal 使用下面的指令前，建议先进入项目目录：

```bash
cd /path/to/summaryassist
```

## 一、最常用指令

### 1. 自动回复总开关

```bash
# 启动自动回复
./autoreply/control.sh start

# 停止自动回复
./autoreply/control.sh stop

# 查看自动回复是否开启
./autoreply/control.sh status
```

执行 `stop` 后，cron 任务仍然存在，但不会读取微信、调用 Qwen 或发送消息。

### 2. 普通自动运行

```bash
# 安装或重新安装 cron
./autoreply/install_cron.sh

# 查看 cron
crontab -l

# 删除自动回复 cron
crontab -l | grep -v '# summaryassist-autoreply-poll' | crontab -
```

启用开关并安装 cron 后，不需要手动输入其他指令。默认空闲状态每 300 秒检查一次；
发现新会话活动后切换为每 10 秒检查一次，连续 30 次没有活动后恢复空闲状态。

### 3. 立即执行

```bash
# 立即抓取消息并生成草稿，但不发送
./autoreply/now.sh

# 立即抓取、生成，并真实回复指定联系人
./autoreply/now.sh --send --contact "联系人显示名或 username"
```

例子：

```bash
./autoreply/now.sh --send --contact "联系人显示名"
```

参数：

| 参数 | 作用 | 出现位置 |
| --- | --- | --- |
| `--send` | 允许真实发送；不添加时只生成草稿 | `autoreply/run_now.py` |
| `--contact` | 指定唯一联系人，防止意外群发 | `autoreply/run_now.py` |

立即执行成功后，空闲计时会重新开始。

### 4. 状态、日志与诊断

```bash
# 一键检查开关、cron、调度状态、AppleScript 权限和 Qwen
./autoreply/diagnose.sh

# 实时查看自动运行日志
tail -f autoreply/logs/poll.log

# 查看最近 50 行日志
tail -n 50 autoreply/logs/poll.log

# 查看当前调度器状态
cat autoreply/scheduler_state.json
```

### 5. 权限与 AppleScript 测试

```bash
# 触发权限请求并打开 macOS 权限设置
./autoreply/request_permissions.sh

# 测试完整 AppleScript 权限链路，不发送消息
./autoreply/test_applescript.sh

# 单独测试 AppleScript 是否能控制微信
osascript sendwechat.scpt --test-access

# 真实发送一条测试消息
osascript sendwechat.scpt "联系人显示名" "测试消息"
```

最后一条指令会真实发送微信消息。

实际发送链路：

```text
Python -> /usr/bin/osascript -> sendwechat.scpt -> System Events -> WeChat
```

### 6. Qwen 与回复草稿

```bash
# 检查 Qwen 服务是否在线
curl http://127.0.0.1:8080/v1/models

# 手动为所有 pending 消息生成回复草稿
./.venv/bin/python -m autoreply.generate_replies

# 预览全部等待发送的最新草稿，不真实发送
./.venv/bin/python -m autoreply.send_replies

# 预览指定联系人的草稿
./.venv/bin/python -m autoreply.send_replies --contact "联系人显示名"

# 发送指定联系人已有的最新草稿
./.venv/bin/python -m autoreply.send_replies --send --contact "联系人显示名"
```

这里的参数由 `autoreply/send_replies.py` 定义：

| 参数 | 作用 |
| --- | --- |
| `--send` | 真实发送已有草稿；不添加时只预览 |
| `--contact` | 只预览或发送指定联系人 |

注意：直接运行 Python 模块时不会自动读取 `autoreply/config.env`。需要临时使用其中参数时：

```bash
set -a
source autoreply/config.env
set +a
./.venv/bin/python -m autoreply.generate_replies
```

`run_poll.sh` 和 `now.sh` 会自动读取 `config.env`。

### 7. 运行测试

```bash
./.venv/bin/python -m unittest discover -s autoreply -p 'test_*.py'
```

## 二、主要参数

日常修改参数时，编辑：

```text
autoreply/config.env
```

当前已配置参数：

| 参数 | 当前值 | 作用 | 读取参数的源码文件 |
| --- | ---: | --- | --- |
| `AUTOREPLY_ALLOWED_SYSTEM_USERNAMES` | 空 | 允许参与测试的系统会话 username；正常使用应保持为空 | `autoreply/poll_unread.py` |
| `AUTOREPLY_IDLE_INTERVAL_SECONDS` | `300` | 空闲状态抓取间隔，单位为秒 | `autoreply/adaptive_scheduler.py` |
| `AUTOREPLY_IDLE_WAKE_EARLY_TOLERANCE_SECONDS` | `5` | 允许 cron 比目标时间提前几秒唤醒，避免一分钟变两分钟 | `autoreply/adaptive_scheduler.py` |
| `AUTOREPLY_AI_MAX_HISTORY_TURNS` | `24` | 每位联系人最多带给 Qwen 的历史对话条数 | `autoreply/generate_replies.py` |
| `AUTOREPLY_AI_MAX_TOKENS` | `1024` | Qwen 单次回复的最大输出 token 数 | `autoreply/generate_replies.py` |

支持但目前没有写入 `config.env` 的参数：

| 参数 | 程序默认值 | 作用 | 读取参数的源码文件 |
| --- | ---: | --- | --- |
| `AUTOREPLY_ACTIVE_INTERVAL_SECONDS` | `10` | 活跃状态抓取间隔，单位为秒 | `autoreply/adaptive_scheduler.py` |
| `AUTOREPLY_EMPTY_POLLS_TO_IDLE` | `30` | 连续多少次无活动后恢复空闲状态 | `autoreply/adaptive_scheduler.py` |
| `AUTOREPLY_SCHEDULER_STATE` | `autoreply/scheduler_state.json` | 调度状态文件位置 | `autoreply/adaptive_scheduler.py` |
| `AUTOREPLY_AI_BASE_URL` | `http://127.0.0.1:8080/v1` | 本地 Qwen OpenAI 兼容 API 地址 | `autoreply/generate_replies.py` |
| `AUTOREPLY_AI_MODEL` | 空，必须由使用者配置 | 调用的模型名称或路径 | `autoreply/generate_replies.py` |
| `AUTOREPLY_DB_PATH` | `autoreply/unread_messages.sqlite3` | SQLite 数据库位置 | `autoreply/poll_unread.py` |
| `AUTOREPLY_WECHAT_CLI` | `wechat-cli` | wechat-cli 命令位置 | `autoreply/poll_unread.py` |
| `AUTOREPLY_HISTORY_LIMIT` | `1000` | 单次联系人历史消息读取上限 | `autoreply/poll_unread.py` |
| `AUTOREPLY_INITIAL_LOOKBACK_SECONDS` | `600` | 无联系人游标时向前读取的秒数 | `autoreply/poll_unread.py` |

修改参数示例：

```bash
# 将空闲抓取间隔设为默认 300 秒
AUTOREPLY_IDLE_INTERVAL_SECONDS=300

# 将每位联系人的记忆扩大为最近 40 条
AUTOREPLY_AI_MAX_HISTORY_TURNS=40

# 将 Qwen 最大输出扩大为 2048 tokens
AUTOREPLY_AI_MAX_TOKENS=2048
```

修改 `config.env` 后，下一次 cron 或 `now.sh` 执行时会读取新值。

## 三、配置本地大模型

本项目不会附带 Qwen 或其他大模型。clone 仓库后，使用者需要自己下载模型并启动本地
OpenAI 兼容 API 服务。

基本步骤：

1. 选择一个可本地运行的模型。
2. 将模型下载到自己的机器，例如 `/absolute/path/to/your/local/model`。
3. 用 MLX、llama.cpp、vLLM 或其他工具启动 OpenAI 兼容接口。
4. 确认服务在线：

   ```bash
   curl http://127.0.0.1:8080/v1/models
   ```

5. 在 `autoreply/config.env` 中填写：

   ```bash
   AUTOREPLY_AI_BASE_URL=http://127.0.0.1:8080/v1
   AUTOREPLY_AI_MODEL=/absolute/path/to/your/local/model
   ```

## 四、Prompt 与联系人记忆

完整系统 Prompt 位于：

```text
autoreply/generate_replies.py
```

具体位置：

- `SYSTEM_PROMPT`：回复风格、安全规则和输出格式。
- `build_user_prompt()`：包装联系人名字和本次未读消息。
- `conversation_history()`：读取同一联系人的历史对话。
- `call_qwen()`：设置模型、温度和最大输出 token。

查看当前系统 Prompt：

```bash
sed -n '31,60p' autoreply/generate_replies.py
```

查看最近一次实际保存的 Prompt：

```bash
sqlite3 autoreply/unread_messages.sqlite3 \
  "SELECT system_prompt, user_prompt FROM reply_drafts ORDER BY id DESC LIMIT 1;"
```

每位联系人的记忆独立保存在数据库 `conversation_turns` 表中。只有回复成功发送后，才会写入记忆。

## 五、数据库查看指令

数据库文件：

```text
autoreply/unread_messages.sqlite3
```

查看数据库内的表：

```bash
sqlite3 autoreply/unread_messages.sqlite3 ".tables"
```

查看最近收到的消息：

```bash
sqlite3 -header -column autoreply/unread_messages.sqlite3 \
  "SELECT id, contact_name, message_text, message_type, status, collected_at
   FROM unread_messages ORDER BY id DESC LIMIT 20;"
```

查看最近生成或发送的回复：

```bash
sqlite3 -header -column autoreply/unread_messages.sqlite3 \
  "SELECT id, contact_name, reply_text, status, created_at
   FROM reply_drafts ORDER BY id DESC LIMIT 20;"
```

查看各联系人的记忆条数：

```bash
sqlite3 -header -column autoreply/unread_messages.sqlite3 \
  "SELECT contact_username, COUNT(*) AS turns
   FROM conversation_turns GROUP BY contact_username;"
```

查看会话分类及忽略原因：

```bash
sqlite3 -header -column autoreply/unread_messages.sqlite3 \
  "SELECT contact_name, category, eligible_for_autoreply, exclusion_reason
   FROM session_audit ORDER BY contact_name;"
```

主要数据库表：

| 表名 | 内容 |
| --- | --- |
| `unread_messages` | 收到的新消息、去重指纹和处理状态 |
| `reply_drafts` | Qwen 回复、完整 Prompt 和发送状态 |
| `reply_draft_messages` | 回复草稿与原消息的关联 |
| `conversation_turns` | 按联系人隔离的连续对话记忆 |
| `contact_cursors` | 每位联系人的历史读取游标 |
| `session_audit` | 群聊、公众号、系统会话和私人联系人分类 |

| 文件 | 作用 |

## 七、正常自动回复流程

```text
cron 每分钟唤醒
  -> run_poll.sh 检查总开关和运行锁
  -> adaptive_scheduler.py 判断现在是否需要抓取
  -> poll_unread.py 调用 wechat-cli new-messages 和 history
  -> 过滤群聊、公众号和系统会话
  -> 消息写入 SQLite，并使用 fingerprint 去重
  -> generate_replies.py 按联系人读取消息和历史记忆
  -> 调用本地 Qwen 生成一条或多条回复
  -> send_replies.py 调用 sendwechat.scpt
  -> 验证发送成功
  -> 更新消息状态并写入该联系人的对话记忆
```
